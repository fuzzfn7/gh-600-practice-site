Exam map

Six domains, weighted by risk and control.

Use this as the top-level map. Domain 2 deserves the most study time because it is the most GitHub-specific.

0115–20%

Prepare agent architecture & SDLC processes

Fit agents into GitHub-driven SDLC without turning planning into unchecked action.

Skills measured
  • Identify agent-performed SDLC steps
  • Mitigate agent anti-patterns
  • Define inputs, outputs, success criteria
  • Separate planning, reasoning, and action
  • Require plan checks before action
  • Produce inspectable artifacts
0220–25%

Implement tool use & environment interaction

Choose, scope, and operate tools/MCP/dev environments safely.

Skills measured
  • Identify required tools
  • Configure tool permissions
  • Add MCP servers
  • Configure MCP registries and allow lists
  • Scope agents to repos/branches
  • Invoke agents from CI
  • Handle retries, rollbacks, escalation, traceability
0310–15%

Manage memory, state & execution

Keep durable state without letting stale context or drift corrupt work.

Skills measured
  • Choose memory strategy
  • Scope memory to relevant info
  • Define expiration/pruning/reset
  • Capture progress and decisions
  • Resume without repeating/diverging
  • Prevent stale/conflicting context
0415–20%

Perform evaluation, error analysis & tuning

Judge agent outputs with signals, traces, scans, logs, and artifacts.

Skills measured
  • Define success criteria
  • Identify qualitative/quantitative signals
  • Use automated scanning tools
  • Analyze logs/plans/traces/artifacts
  • Classify reasoning/tool/context/environment failures
  • Tune instructions, workflows, memory, and tool access
0515–20%

Orchestrate multi-agent coordination

Coordinate multiple agents without conflicts, duplicated work, or lost auditability.

Skills measured
  • Apply orchestration patterns
  • Configure isolated parallel execution
  • Resolve overlapping changes/duplicated effort/contradictory outputs
  • Produce review/audit artifacts
  • Recover stalled/degraded agents
  • Add/update/replace/retire agents cleanly
0610–15%

Implement guardrails & accountability

Right-size autonomy and approvals by risk, compliance, and irreversibility.

Skills measured
  • Classify actions by risk
  • Assign autonomy levels
  • Identify actions requiring human judgment
  • Block policy violations
  • Enforce least-privilege execution
  • Require explicit authorization for irreversible/compliance-sensitive actions

Hands-on labs

Build what the test describes.

These are not required by the exam vendor. They are how you turn abstract guide bullets into muscle memory.

35 min

Lab 1 — Agent plan gate

Goal: Create a repo issue template where an agent must output inputs, plan, files touched, approval needed, and success criteria before changing code.

Proof: A PR or markdown artifact showing the plan gate and a rejected unsafe plan example.

45 min

Lab 2 — MCP allow-list sketch

Goal: Draft an MCP config that exposes only issue/search/read-only repo tools to a research agent, then separately allows branch/PR tools to a coding agent.

Proof: Two tool-permission matrices and a note explaining why each permission exists.

50 min

Lab 3 — CI agent invocation

Goal: Design a GitHub Actions workflow that runs an agent/evaluator on pull requests and uploads logs/artifacts.

Proof: Workflow YAML plus sample artifact names and failure/escalation behavior.

40 min

Lab 4 — Eval rubric

Goal: Write an evaluation rubric for an agent that edits a workflow file: expected output, scans/tests, qualitative review, and root-cause buckets.

Proof: A scorecard with pass/fail criteria and examples of reasoning/tool/context/environment failures.

45 min

Lab 5 — Multi-agent conflict drill

Goal: Simulate two workers editing overlapping files. Define isolation, handoff, conflict detection, and resolution steps.

Proof: A post-hoc analysis artifact showing decisions, conflicts, and final accepted path.

30 min

Lab 6 — Autonomy matrix

Goal: Classify actions into auto, notify, approval, and blocked categories for a client-facing agent workflow.

Proof: A risk matrix with examples of irreversible/compliance-sensitive actions and approval rules.

Glossary

GitHub / Copilot / agentic AI terms.

Agent architecture

The design of agent responsibilities, inputs, outputs, tools, memory, autonomy, and review gates.

Branch-based scope

Restricting an agent’s code changes to a specific branch so work stays reviewable and isolated.

Copilot custom agent

A GitHub/Copilot agent customized with instructions, setup, tools, and scoped behavior.

Copilot setup steps

Configuration that prepares an agent environment before it works, such as dependencies or repo-specific setup.

GitHub control plane

Using GitHub issues, branches, PRs, Actions, checks, and artifacts as the system of record for agent work.

Operational signal

Evidence from execution that shows health or failure: logs, exit codes, artifacts, scans, status checks.

Responsible AI guardrail

A policy or control that prevents unsafe, non-compliant, or unaccountable agent behavior.

Traceability

The ability to reconstruct who/what acted, why, with which tools, and what changed.
